# Modelo de Segurança e Governança

- Utilize autenticação e autorização para acesso aos endpoints e dashboards
- Restrinja permissões das funções serverless ao mínimo necessário
- Proteja dados sensíveis no banco e em trânsito (TLS)
- Gerencie segredos via serviço seguro do provedor cloud (ex: Azure Key Vault, AWS Secrets Manager)
- Implemente RBAC/IAM para administradores e operadores
- Documente todas as políticas e configurações aplicadas

> Atualize este arquivo conforme as práticas e ferramentas do seu provedor cloud.
