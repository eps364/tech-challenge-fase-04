# RF-06 - Disponibilizar exemplos de request/response atualizados na documentacao

## Objetivo
Manter a documentacao da API atualizada e coerente com os fluxos de negocio e com o que sera implementado.

## Escopo funcional
- Atualizar exemplos de `POST /avaliacao`.
- Atualizar exemplos de notificacao de urgencia.
- Atualizar exemplos de relatorio semanal.
- Garantir consistencia entre README da API, exemplos e diagramas.

## O que sera realizado
1. Revisar contratos em `docs/API/README.md`.
2. Revisar payloads em `docs/API/exemplos.md`.
3. Validar alinhamento com `docs/diagrams/architecture.mmd` e `docs/diagrams/flows.mmd`.
4. Registrar premissas quando houver lacunas no enunciado.

## Criterios de aceite
- Os exemplos refletem os RF-01 a RF-05.
- Nao ha contradicao entre arquivos de documentacao.
- A equipe avaliadora consegue reproduzir entendimento da API apenas pela documentacao.

## Evidencias esperadas
- Secoes atualizadas e coerentes em `docs/API/README.md` e `docs/API/exemplos.md`.
- Referencia cruzada no documento principal (`docs/document.md`).
